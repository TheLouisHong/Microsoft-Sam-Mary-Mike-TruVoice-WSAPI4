Telephony Control Sample

This sample telephony control shows you how to write your own telephony controls. It provides the source code used to produce the telephony controls in the Microsoft Speech API. You can modify the controls to suit your needs, or use them as templates for your own custom controls.

To test the telephony controls in the sample, you must:

  1. Run Visual C++ and edit the sample source code.
  2. Make sure the resulting telctl.dll is compiled into your Speech SDK telephony-files directory. This is usually "c:\program files\ Microsoft speech SDK\tel". It will overwrite an existing file.
  3. Compile the sample, preferably as debug.
  4. You may also want to have a debug version of spchwrap.lib built and linked into telctl.dll, since spchwrap.lib provides major functionality for the telephony controls.
  5. Set the project to run the telephony control editor when debugging. The telephony control editor is usually located in "c:\program files\Microsoft speech SDk\tel\tel.exe"
  6. Place a break-point in the code you wish to test. A good place to start would be DllGetClassObject() in cf.cpp.
  7. Start VC++ debugging. This will run tel.exe.
  8. In Tel.exe, choose the File\New menu item.
  9. In the new-file dialog box, choose any telephony control type starting with "Sample".
  10. Press OK
  11. If debugging is working properly and a breakpoint was placed in DllGetClassObject(), then the breakpoint will be hit now. You can debug from here to see how a telephony control works.
