Audio Collection Tool

The audio collection tool allows you to easily collect audio recordings from a number of different users.

Producing a phrase file

First, you need to determine what the speakers will record. You do this by listing out hundreds or even thousands of phrases/sentences that should be spoken. An individual speaker might only record a subset of the phrases, so that each speaker records a different subset.

The file format for a phrase file is the text “[Sentences]” at the top of the file followed by a sequentially ordered list if phrases.

[Sentences]

1=Hello

2=Bye

3=Ford

4=This is a test

5=Check this out

Avoid using punctuation, symbols, and numbers in the file. You can write out numbers as words though.

Recording speakers

To record the speakers, run the Audio Collection Tool.

Filling in the administrative page

When you first run COLLECT.EXE, you will see an administrative page that the operator should fill in.

You need to fill in the following information:

  • Phrase path and filename -- Write the full path to the “phrase.ini” file. If it’s in the same directory as COLLECT.EXE then you should type in “.\phrase.ini”
  • Users directory -- Find a disk, either on the local machine or server, that has about 500 megabytes free. Create a directory on it. Type this directory into the field. (Ex: “c:\users”) Every user that is recorded will have a sub-directory created by COLLECT.EXE. (Ex: User number 52 will have his/her recordings written in “c:\users\u000052”)
  • Microphone -- Type in the name of the microphone being recorded with.
  • Sound card -- Type in the name of the sound card being recorded with.
  • Language -- Type in the name of the language being recorded.
  • Recording institution -- Type in your company’s name.
  • Sampling rate -- Type in the sampling rate: “8000” for phone data. Use “11025”, “16000”, or “22050” for PC recognition data depending upon the speech engine. If you don't know what sampling rate to use, type in 22050, and let the accuracy test application downsample the data for you if it's the wrong sampling rate.
  • User Number -- Start out with user number 1. Every time you run this program, the user number will automatically be incremented, so you don’t have to do any extra work.
  • Start at phrase number -- Start out with phrase number 1. Every time you run this program, the phrase number will be incremented to the last one recorded by the last user. This insures that all of the phrases/sentences are recorded. You will not have to modify this field after the first time.
  • Number of phrases to speak -- You probably want this to be 100 since most people tire after speaking more than 100 phrases.
  • Speaking style -- Fill in the eight fields with some common dialects for the language. The user will be able to select from one of these.

When you press “Next”, a dialog asking for user information will start up.

You only have to fill this in once, because the information will be stored away and used the next time that the application is run.

Have the user fill in user-information

The end-user should fill out the “User Information” dialog. The fields are:

  • Name
  • Gender
  • Age -- Users may not want to fill out their exact age, so they can just type in a number that’s approximate.
  • Phone number -- Users may not want to type in their phone number. We would like this so we can contact them at some later point if we’d like more speech data.
  • Beginning/experienced/advanced user -- This just gives us a feel for how comfortable they are with the computer.
  • Speaking style -- A user can select one of the eight pre-defined ones, or type in something different.
  • Comments -- If you’ve got any special comments to make.

All of this information is saved in the user directory on disk.

When the user presses, “Next”, he/she goes onto the recording session.

The recording session

The Audio Collection Tool will first bring up a dialog of instructions for the user. When the user presses “Next”, he/she will begin recording.

The collection program will occupy the entire screen, and ask the user to speak one of the phrases. When the user speaks a phrase, it will be played back to him. He/she will then be asked if the computer recorded the correct phrase. If the user presses “No”, he/she returns to recording the old phrase. If he presses “Yes” then he/she moves onto the next phrase.

Some common problems at this point are:

  • The user speaks too loudly/quietly -- The computer will re-adjust the recording volume control (if it can) and have the user speak the phrase over.
  • The background noise is too loud -- The user should move closer to the microphone or you should find a way of making the room quieter.
  • Background noise is recorded -- If you make a somewhat loud sound, the computer may think that it heard speech.
  • The computer records only half a phrase -- The user might be leaving too much of a gap between words.

The user can pause/resume recording with the two radio buttons.

You should have someone around for the recording of the first few phrases just to make sure that the volunteer understands what he/she is supposed to do. After that, you can leave. The program will inform the user when he/she is all done.

When the user is done

The program informs the user when he/she is all done, at which point he/she can call you on the phone.

How the data is written

All of the data is written to the directory specific in the administration dialog. (Example: “c:\users”.)

Each user will have a new subdirectory created based upon his/her user number. (Example: User 26 is “c:\users\u0000026”)

Under each user directory, you will find a “user.ini” which stores all of the information about the user and the sentences that he/she spoke. You will also find one wave file per phrase spoken, their names based upon the phrase number. (Example: If the user spoke phrases 400 through 499, you’d see “c:\users\u0000026\s000400.wav” through “c:\users\u0000026\s000499.wav”)

The .wav files are standard format for windows and can be played by any program.
