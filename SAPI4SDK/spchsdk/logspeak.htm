Wave List Editor

The wave list editor allows you to easily:

  • Cut the CPU time text-to-speech spends speaking prompts in your telephony application, by pre-recording common text-to-speech prompts.
  • Replace text-to-speech prompts in your telephony application with recorded speech.

 

To use the Wave List Editor application, you should take the following steps:

  1. Modify your telephony application so that it uses logging. If you are using the telephony wrapper objects, this is fairly easy; You just change a setting in the "logging" dialog box.
  2. Have a number of people call into your telephony application. The number of calls you'll need depends upon the size of your application. You should have enough calls logged that all of the major prompts have been spoken by text-to-speech. Each call will have a log.
  3. Run the Wave List Editor Application.
  4. Click the "Analyze Logged Data" button, and type in the directory where your application's logs (*.stl files) are stored. The software will analyze the logs and determine which phrases/prompts were spoken, and how often each was spoken.
  5. Click the "Edit wave list" button to see the results of the analysis. The window will show all of the currently unrecorded prompts, sorted from the most frequent to the least frequent.
  6. You may not want to record some of the least frequent prompts because the hard drive space used is not worth the CPU savings, especially if the prompts are automatically generated and are not likely to be spoken again. To not record the prompts, select the prompt in the list, and press the "Ignore" button. This will move the prompt to the "Ignored" list. Of course, you can move prompts from the "Ignored" list back onto the record list later.
  7. Determine which prompts you want a real person to record. This will depend upon the needs of your application. To record a prompt, select the prompt in the list, and press "Record Wave". The "Record Wave" dialog will ask for a file name. Be sure to type in the full path and file name. Make sure that the same path will work from your telephony application. When you have typed in the file name, press "Record." A new file will be created, and the wave editor will be brought up to record. Use the wave editor to record the prompt, and modify the recording.
  8. Repeat the record process until you have recorded all the prompts you plan to. What's left will be those prompts that you're willing to have synthesized.
  9. Close the "Edit Wave List" dialog, and press the "Automatically Fill Wave List" button. This will automatically pre-synthesize any "Unrecorded" prompts. (It will not synthesize prompts in the "Ignored" or "Recorded" list.) You must select a text-to-speech voice to use, and type in the directory where to write the wave files. File names will automatically be generated. Press "OK" to start processing.
  10. You can hear the pre-synthesized prompts by returning to the "Edit Wave List" dialog.
  11. Finally, you should run "Convert wave list" from the main dialog to make sure that all your wave files are 8 kHz. If they're not 8 kHz, they won't work over the phone.
  12. To save your wave list, press "Save wave list" and type in the list name. The file is a Unicode text file that you can open with Notepad (under Windows NT). The file format is fairly easy to understand.
  13. For your telephony application to use the recordings, your application needs to load the wave-list text data into memory, and pass it into either ISTTTSLog::WaveAddFromList or ITelInfo::QuickCreate. The file format is compatible with the data formats used by WaveAddFromList or QuickCreate.
  14. To make development quicker, you can save the wave list in your telephony application's project directory. Modify your resources to include the wave-list file as type "TEXT", and have your project load the resource, and pass it into WaveAddFromList or QuickCreate. If you are using the C++ wrappers, you can use an overloaded version of WaveAddFromList or QuickCreate that automatically loads the data from a resource.
