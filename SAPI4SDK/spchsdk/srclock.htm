A clock you can talk to

The Speech Recognition Clock Demo (SRCLK in the samples\clock directory) demonstrates the use of the Speech Recognition and Text-To-Speech Application interface. Upon recognition of certain voice commands, SRCLK responds audibly with the current time or date using text-to-speech. SRCLK was built using Microsoft Visual C++ 4.0.

To use SRCLK, run the application and ask "What is the date?" or "What time is it?" SRCLK will respond with the correct date or time. You can also ask for the time in other cities such as London, Stockholm, or Tokyo.

Low-Priority Mode Drivers

The Speech Recognition Clock demo uses both speech recognition (SR) AND text-to-speech (TTS). If the wave input driver being used does not support low-priority mode, there are certain functions which must be performed before text-to-speech will work properly.

Here's why: When the wave input device is opened in low priority mode by a process (for example, speech recognition), that device is available to any process that subsequently requests it (for example, wave output for text-to-speech). When the requesting process relinquishes the device, it is again available to the original process. In a driver that doesn't support-low-priority mode, the original process (e.g., speech recognition) gains exclusive use of the wave-in device and does not relinquish it until it is finished. Therefore, the requesting process (text-to-speech) cannot open the device.

SRCLK employs a workaround for handling drivers that don't support low-priority mode. It deactivates speech recognition when text-to-speech requires the device and reactivate speech recognition when text-to-speech is done.

Notes

SRCLK reads a data file (srclk.dat) which contains time zone information for various cities worldwide. You can modify this data file to add or delete cities. The syntax is simply the city name followed by a corresponding time zone bias separated by a comma. The time zone bias value is the number of hours the city is away from Greenwich Mean Time (GMT).

For example, London is the same time zone as GMT so its bias value is 0. Seattle is Pacific time which is eight hours behind GMT so its bias value is -8, etc.

SRCLK contains 6 source modules:

  • init.cpp - Windows initialization code
  • showdib.cpp - Loads and displays clock bitmap
  • vcmd.cpp - Contains code for interacting with the SR and TTS interfaces
  • srclk.cpp - Main procedure and miscellaneous functions for the clock
  • clkhands.cpp - Contains code for initializing and drawing the clock hands.
  • srclkdlg.cpp - Contains code for handling the Clock Options and About dialogs

Miscellaneous required files:

  • srclk.dat - Text file containing the city and time zone information file
  • srclk.bmp - Bitmap file containing the clock's graphical face

SRCLK builds a VCMD list for the cities it will recognize. If Microsoft Voice is running the user can find out what the names of these cities are by saying "What can I say" and clicking on <CITYNAME>.

You need this to run:

  • Command & Control Speech Recognition Engine
  • Text-to-speech engine
  • A navigator application must be running. You can install Microsoft Voice, or just run Feedback.
