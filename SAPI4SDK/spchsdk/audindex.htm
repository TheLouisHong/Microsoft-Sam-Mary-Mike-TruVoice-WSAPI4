Audio Indexer

The Audio Indexer sample uses continuous dictation to index speech audio recordings. It allows users to do keyword searches on the index, and easily jump to the place in the recording where the keywords were used. Of course, the indexing is only as accurate as the speech recognizer.

The source code demonstrates how to use ISRResGraphEx, the file audio source object, and controlling an engine's thread priority.

To run the demo:

  1. You should pre-record the sample you wish to index. Make sure to leave a few seconds of silence at the end of the recording so the recognizer will index the last phrase. The recording should be 16-bit mono, and depending upon the recognizer, either 16 kHz or 22 kHz sampling rate. Telephone recordings use 8 kHz. If you record with the wrong sampling rate, you can up/down-sample with the wave editor.
  2. Run it
  3. Press "Open Wave" and, using the file-open dialog box, select the wave file you wish to use.
  4. If you haven't indexed the file before, you will be asked to index it now. Press "Yes." (If you're not asked, press the "Index Wave" button.)
  5. In the "Select Speech Recognition" dialog box:
  6. Select the speech recognition engine to use.
  7. If the audio was recorded by a speaker who has trained on your system, then type his/her name into the speaker field. This will make indexing more accurate.
  8. Select the processing time, indicating how rapidly the audio will be passed to the recognizer. A rate of 1/4 means that 1 hour of audio will take 4 hours to process, but will consume 1/4 the CPU bandwidth normally used by the recognizer.
  9. Select the recognizer's accuracy level to use. This affects how much CPU is used, but not how long it takes to process. You usually want maximum accuracy, which takes the most CPU.
  10. Press OK.
  11. The recognizer will be started and will process the audio data. The "Progress" field shows you how far the indexing has gotten. When indexing is finished, the dialog will automatically close and return you to the "Audio Indexer" window.
  12. In the "Audio Indexer" window, to play the recording, press "Play". The currently indexed word will be displayed as the audio plays.
  13. To find a word, type it into the "Search for" field. Only type one word in. Press "Find next" or "Find previous" to find the next/previous occurrence of the word.
  14. By default, the search not only looks through the recognizer's top recognition result for the word, but also through alternative guesses, just in case the top recognition result was wrong. You can disable this by un-checking the "Search using alternatives also" check-box.
  15. That's it.
